# Addendum — The Markdown Web brief

Downstream context that earned a place but does not belong in the 1-2 page brief — for the PRD, positioning, and architecture work that follows.

## Competitive / defensibility context (cut from brief at user's request)
The brief states the differentiated position (the unclaimed intersection) but omits the defensibility caveat:
- **Plumbing is commoditizing:** markdown hosting, content negotiation, and md/HTML dual-serve are being given away (Cloudflare "Markdown for Agents," Vercel, Mintlify) and owned for publishing (Hosted.md, Quartz, Obsidian Publish). Do not position as "host my markdown" or "serve md to agents."
- **The native client is attackable** by entrants who already own client + agent: Perplexity Comet, Browser Company Dia, ChatGPT Atlas, Google A2UI / Generative UI.
- **Therefore the moat is non-technical:** category ownership ("The Markdown Web" / themarkdownweb.com), the publish↔read loop, an opinionated beautiful client, community, and the personal-vault wedge (useful with zero network).
