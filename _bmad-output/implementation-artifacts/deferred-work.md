- **DEFERRED BY DECISION (Option 2 — naethyn, 2026-06-21): true same-URL `Accept` content negotiation.** Azure Static Web Apps route rules cannot branch on the request `Accept` header (`rewrite`/`redirect` are path-only — a documented platform limitation), so a `text/markdown` request to a page URL `/<slug>` cannot be forwarded to the Function purely via `staticwebapp.config.json`. Pragmatic resolution shipped: the page URL `/<slug>` stays **pure static HTML** (fast static host, live site untouched) and **raw markdown is exposed NOW at the Function endpoint `/api/negotiate/<slug>`** (`text/markdown; charset=utf-8` + `Vary: Accept`, byte-faithful, shipped in the deploy package). The native client (Epic 3, Story 3.2) — the only consumer — fetches markdown from `/api/negotiate/<slug>`, so the same-URL ideal is not blocking for the MVP. REVISIT when Epic 3 lands: if true one-URL is required, options are (a) front page routes with the Function and proxy/redirect the HTML branch to the static asset, or (b) an Azure Front Door / edge rule that can branch on `Accept`. Story 2.7 AC1/AC7 document this; the original ACs describe the deferred ideal. [web/public/staticwebapp.config.json; api/negotiate/index.mjs route `negotiate/{*slug}`]
